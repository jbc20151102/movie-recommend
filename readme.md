基于 MovieLens 100K 数据集实现的混合推荐系统，融合 Item-Based 协同过滤（ItemCF）和内容特征推荐，通过网格搜索优化融合权重 α，最终输出 Top-K 电影推荐结果。
使用内核：python3.13.9

##  1.技术栈与环境要求
### 技术栈
| 类别         | 核心库/工具                          | 功能说明                     |
|--------------|--------------------------------------|------------------------------|

| 机器学习     | scipy、scikit-learn                  | 相似度计算、模型评估、网格搜索 |
| 可视化       | matplotlib、seaborn                  | 多维度英文图表生成（300dpi） |
| 缓存/文件操作 | pickle、os                           | 缓存读写、目录自动创建       |

### 2. 数据集准备
1. 下载 MovieLens 100K 数据集：https://grouplens.org/datasets/movielens/100k/
2. 解压后将包内文件放入 `data/ml-100k/` 目录

### 3. 运行项目
直接执行主脚本，自动完成全流程（数据加载→特征工程→推荐→可视化）：
```bash
python src/main.py
```

## 核心功能
1. **数据预处理**：清洗评分数据、提取电影上映年份、过滤小众用户/电影、拆分训练集/测试集；
2. **特征工程**：
   - 构建用户-电影评分矩阵（协同过滤基础）；
   - 构建电影内容特征矩阵（类型 + 上映年份/年代离散化 + 归一化）；
3. **相似度计算**：
   - ItemCF 相似度：基于评分矩阵的余弦相似度（中心化处理减少用户偏差）；
   - 内容相似度：基于电影类型/年份特征的余弦相似度；
4. **推荐算法**：
   - 纯 ItemCF 推荐：基于物品相似度的评分预测；
   - 纯内容推荐：基于电影内容特征的相似度推荐；
   - 混合推荐：融合 ItemCF 和内容推荐，权重 α 控制 ItemCF 占比（1-α 为内容推荐占比）；
5. **模型评估**：计算 Precision@K、Recall@K、F1@K，通过网格搜索找到最优 α；
6. **结果输出**：为指定用户生成 Top-10 推荐结果，保存到报告文件。

## 📈 核心算法逻辑
### 1. 混合推荐评分公式
```
混合预测评分 = α × ItemCF预测评分 + (1-α) × 内容推荐预测评分
最终评分 = np.clip(混合预测评分, 0, 5)  # 仅夹紧极端值，保留评分差异
```
- α ∈ [0,1]：α=1为纯ItemCF推荐，α=0为纯内容推荐，网格搜索选F1@K最高值为最优α

### 2. 全局热门推荐得分
```
全局得分 = 0.7 × 电影平均评分 + 0.3 × 归一化评分人数（缩放至0-5）
```
- 平衡评分质量与大众热度，避免「高分冷门」或「低分热门」推荐

## 核心算法细节
### 1. 混合推荐评分公式
```
混合评分 = α * ItemCF预测评分 + (1-α) * 内容推荐预测评分
```
- α 范围 0~1：α=1 等价纯 ItemCF，α=0 等价纯内容推荐；
- 评分预测后做区间夹紧（2.8~4.8），避免极端低分/高分。

### 2. 评估指标计算
- Precision@K：推荐列表中用户喜欢（评分≥4）的电影占比；
- Recall@K：用户喜欢的电影中被推荐的占比；
- F1@K：Precision 和 Recall 的调和平均（作为最优 α 选择依据）。

### 关键参数说明
- `K`：推荐 Top-K 数量（默认 10）；
- `TARGET_USER`：目标推荐用户 ID（默认 200）；
- `sample_users`：评估时采样用户数（默认 100，提速用）；
- `alphas`：网格搜索的 α 范围（默认 [0.0,0.2,0.4,0.6,0.8,0.95,1.0]）。


## 注意事项
1. 若运行时提示文件不存在，检查 `data/ml-100k/` 目录下是否有完整数据集；
2. 调整 `sample_users` 参数可平衡评估速度与精度（值越大越精准，耗时越长）；
3. 推荐评分做了人工校准（偏移 + 夹紧），目的是避免低分推荐，提升用户体验。
